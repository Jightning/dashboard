# Principles

1. Don't overengineer: Simple beats complex, and don't reinvent the wheel
2. Throw errors: Fail fast when preconditions aren't met
3. Fix root causes: Address the underlying issue, not just symptoms
4. Collaborative process: Work with user and ask questions to identify most efficient solution
5. The user is not always right: If a request is faulty or asks for something illogical, ask the user if they prefer an improved version
